10.1 Comments

In Python, the key for a comment is #.

# This line is a comment.
print("This line is not a comment.")
This line is not a comment.

10.2 Two equality signs

In Python (and many other programming languages), there are assignments such as x = 0, which says that x should from now on have the value 0, and x == 0. The latter is a condition, which results in True if x is indeed 0, and `False otherwise.

10.3 User input

Sometimes, when operating on the command line, one wants to ask for user input. This works as follows:

name = input("What is your name?")
print(f"Hello, {name}!")

10.5 dot-notation

Here is another very useful notation convention, known as dot-notation: For a function str.something(s,...), where s is of type str, you can as well write s.something(). For example, s.strip() is the same as str.strip(s).

10.6 git and github

These course notes are organized using the version control system git. If this does not mean anything to you, don’t mind. At the moment, this only means that you need to install git on your system, and run in the folder where you want to have the course notes

git clone https://github.com/pfaffelh/python_for_data/

With this command, you are cloning the repository of the course notes from github. You can also look at this repository online.

There is a good chance that the course notes will be updated during the upcoming weeks. In order to obtain the most current version, you need to run

git pull

There is (at least) one catch for using git: If you are changing one of the course files locally on your system, but the file has updated, git pull realizes the different versions and rejects to do git pull. The best way out is to do all changes to the files in the folder myFiles, or only change the files ...ipynb. All these files are excluded from any update with git pull.

We recommend that you do the following:

  1. Clone the repository of these notes using
git clone https://github.com/pfaffelh/python_for_data

Hit cd python_for_data. 2. Create a virtual environment using python -m venv venv or python3 -m venv venv. 3. Activate this environment by source venv/bin/activate, 4. Install the libraries using pip3 install -r requirements.txt (or pip install -r requirements.txt).
